Documentación de API REST

La API REST del Sistema de Gestión te permite integrar con otros sistemas y automatizar procesos.

Autenticación​

Obtener Token de Acceso​

POST /api/auth/login
Content-Type: application/json

{
  "email": "usuario@empresa.com",
  "password": "tu_password"
}

Respuesta:

{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "name": "Usuario",
    "email": "usuario@empresa.com",
    "role": "admin"
  }
}

Usar Token en Requests​

GET /api/stock/products
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Endpoints de Stock​

Obtener Productos​

GET /api/stock/products

Parámetros de consulta:

  • page: Número de página (opcional)
  • limit: Límite de resultados (opcional)
  • category: Filtrar por categoría (opcional)
  • search: Buscar por nombre (opcional)

Respuesta:

{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "Producto 1",
      "code": "PROD001",
      "category": "Categoría 1",
      "stock": 100,
      "price": 25.50
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 1,
    "pages": 1
  }
}

Crear Producto​

POST /api/stock/products
Content-Type: application/json

{
  "name": "Nuevo Producto",
  "code": "PROD002",
  "category_id": 1,
  "price": 30.00,
  "stock": 50
}

Actualizar Producto​

PUT /api/stock/products/1
Content-Type: application/json

{
  "name": "Producto Actualizado",
  "price": 35.00
}

Eliminar Producto​

DELETE /api/stock/products/1

Endpoints de Finance​

Obtener Transacciones​

GET /api/finance/transactions

Parámetros de consulta:

  • type: Tipo de transacción (income/expense)
  • date_from: Fecha desde (YYYY-MM-DD)
  • date_to: Fecha hasta (YYYY-MM-DD)

Crear Transacción​

POST /api/finance/transactions
Content-Type: application/json

{
  "type": "income",
  "amount": 1000.00,
  "description": "Venta de productos",
  "date": "2024-01-15",
  "category_id": 1
}

Endpoints de Services​

Obtener Servicios​

GET /api/services

Crear Servicio​

POST /api/services
Content-Type: application/json

{
  "client_id": 1,
  "vehicle_id": 1,
  "service_type": "maintenance",
  "description": "Mantenimiento preventivo",
  "scheduled_date": "2024-01-20",
  "technician_id": 1

Endpoints de Usuarios​

Obtener Usuarios​

GET /api/users

Crear Usuario​

POST /api/users
Content-Type: application/json

{
  "name": "Nuevo Usuario",
  "email": "nuevo@empresa.com",
  "password": "password123",
  "role": "employee"
}

Códigos de Estado HTTP​

  • 200 OK: Solicitud exitosa
  • 201 Created: Recurso creado exitosamente
  • 400 Bad Request: Solicitud malformada
  • 401 Unauthorized: No autenticado
  • 403 Forbidden: Sin permisos
  • 404 Not Found: Recurso no encontrado
  • 500 Internal Server Error: Error del servidor

Manejo de Errores​

Estructura de Error​

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Los datos proporcionados no son válidos",
    "details": [
      {
        "field": "email",
        "message": "El email es requerido"
      }
    ]
  }
}

Códigos de Error Comunes​

  • VALIDATION_ERROR: Error de validación de datos
  • AUTHENTICATION_ERROR: Error de autenticación
  • AUTHORIZATION_ERROR: Error de autorización
  • NOT_FOUND: Recurso no encontrado
  • DUPLICATE_ERROR: Recurso duplicado
  • SERVER_ERROR: Error interno del servidor

Límites y Rate Limiting​

Límites de API​

  • Requests por minuto: 100 requests
  • Requests por hora: 1000 requests
  • Tamaño de payload: 10MB máximo
  • Timeout: 30 segundos

Headers de Rate Limiting​

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1640995200

Ejemplos de Uso​

JavaScript/Node.js​

const axios = require('axios');

// Configurar cliente
const api = axios.create({
  baseURL: 'https://tu-dominio.com/api',
  headers: {
    'Authorization': 'Bearer tu_token_aqui'
  }
});

// Obtener productos
async function getProducts() {
  try {
    const response = await api.get('/stock/products');
    console.log(response.data);
  } catch (error) {
    console.error('Error:', error.response.data);
  }
}

// Crear producto
async function createProduct(productData) {
  try {
    const response = await api.post('/stock/products', productData);
    console.log('Producto creado:', response.data);
  } catch (error) {
    console.error('Error:', error.response.data);
  }
}

Python​

import requests

# Configurar cliente
headers = {
    'Authorization': 'Bearer tu_token_aqui',
    'Content-Type': 'application/json'
}

# Obtener productos
def get_products():
    response = requests.get(
        'https://tu-dominio.com/api/stock/products',
        headers=headers
    )
    return response.json()

# Crear producto
def create_product(product_data):
    response = requests.post(
        'https://tu-dominio.com/api/stock/products',
        json=product_data,
        headers=headers
    )
    return response.json()

Consejos para el Uso de la API​

Mejores Prácticas​

  • Usa HTTPS: Siempre usa conexiones seguras
  • Maneja errores: Implementa manejo robusto de errores
  • Cachea respuestas: Implementa caché cuando sea apropiado
  • Monitorea límites: Respeta los límites de rate limiting
  • Valida datos: Valida datos antes de enviar
  • Usa paginación: Para grandes conjuntos de datos
  • Mantén tokens seguros: No expongas tokens en el código
