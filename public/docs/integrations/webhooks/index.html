Documentación de Webhooks

Los webhooks te permiten recibir notificaciones en tiempo real cuando ocurren eventos en el sistema.

Configurar Webhooks​

Crear Webhook​

POST /api/webhooks
Content-Type: application/json

{
  "url": "https://tu-servidor.com/webhook",
  "events": ["product.created", "product.updated"],
  "secret": "tu_secreto_webhook"
}

Eventos Disponibles​

Eventos de Stock​

  • product.created: Producto creado
  • product.updated: Producto actualizado
  • product.deleted: Producto eliminado
  • stock.low: Stock bajo
  • stock.out: Stock agotado

Eventos de Finance​

  • transaction.created: Transacción creada
  • payment.received: Pago recibido
  • payment.overdue: Pago vencido

Eventos de Services​

  • service.created: Servicio creado
  • service.completed: Servicio completado
  • service.cancelled: Servicio cancelado

Eventos de Usuarios​

  • user.created: Usuario creado
  • user.updated: Usuario actualizado
  • user.deleted: Usuario eliminado

Verificar Webhooks​

Verificación de Firma​

const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  
  return signature === `sha256=${expectedSignature}`;
}

Ejemplo en Node.js​

const express = require('express');
const crypto = require('crypto');

app.post('/webhook', (req, res) => {
  const signature = req.headers['x-webhook-signature'];
  const payload = JSON.stringify(req.body);
  const secret = process.env.WEBHOOK_SECRET;
  
  if (!verifyWebhook(payload, signature, secret)) {
    return res.status(401).send('Invalid signature');
  }
  
  const { event, data } = req.body;
  
  switch (event) {
    case 'product.created':
      handleProductCreated(data);
      break;
    case 'product.updated':
      handleProductUpdated(data);
      break;
    default:
      console.log('Unknown event:', event);
  }
  
  res.status(200).send('OK');
});

Manejo de Errores​

Reintentos Automáticos​

El sistema reintenta automáticamente los webhooks fallidos:

  • Primer reintento: 1 minuto después
  • Segundo reintento: 5 minutos después
  • Tercer reintento: 15 minutos después
  • Cuarto reintento: 1 hora después
  • Máximo: 4 reintentos

Códigos de Estado​

  • 200 OK: Webhook procesado exitosamente
  • 400 Bad Request: Payload malformado
  • 401 Unauthorized: Firma inválida
  • 500 Internal Server Error: Error del servidor

Ejemplos de Implementación​

PHP​

<?php
function verifyWebhook($payload, $signature, $secret) {
    $expectedSignature = hash_hmac('sha256', $payload, $secret);
    return hash_equals($signature, "sha256={$expectedSignature}");
}

$payload = file_get_contents('php://input');
$signature = $_SERVER['HTTP_X_WEBHOOK_SIGNATURE'];
$secret = $_ENV['WEBHOOK_SECRET'];

if (!verifyWebhook($payload, $signature, $secret)) {
    http_response_code(401);
    exit('Invalid signature');
}

$data = json_decode($payload, true);
$event = $data['event'];

switch ($event) {
    case 'product.created':
        handleProductCreated($data['data']);
        break;
    default:
        error_log("Unknown event: {$event}");
}

http_response_code(200);
echo 'OK';
?>

Python​

import hashlib
import hmac
import json
from flask import Flask, request

app = Flask(__name__)

def verify_webhook(payload, signature, secret):
    expected_signature = hmac.new(
        secret.encode(),
        payload.encode(),
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(signature, f"sha256={expected_signature}")

Consejos para Webhooks​

Mejores Prácticas​

  • Verifica firmas: Siempre verifica la firma del webhook
  • Maneja errores: Implementa manejo robusto de errores
  • Procesa asíncronamente: Usa colas para procesar webhooks
  • Mantén logs: Registra todas las actividades
  • Monitorea rendimiento: Supervisa el rendimiento de webhooks
  • Usa HTTPS: Siempre usa conexiones seguras
  • Valida datos: Valida los datos recibidos
