Esta sección cubre las configuraciones avanzadas del Sistema de Gestión Empresarial.

Configuración del Sistema​

Variables de Entorno​

El sistema utiliza varias variables de entorno para su configuración:

# Base de datos
DATABASE_URL="postgresql://usuario:password@localhost:5432/sistema_gestion"

# Autenticación
NEXTAUTH_SECRET="tu-secreto-super-seguro"
NEXTAUTH_URL="http://localhost:3000"

# Configuración de la aplicación
NEXT_PUBLIC_APP_URL="http://localhost:3000"
NEXT_PUBLIC_API_URL="http://localhost:3000/api"

# Configuración de email
SMTP_HOST="smtp.gmail.com"
SMTP_PORT="587"
SMTP_USER="tu-email@gmail.com"
SMTP_PASS="tu-password-de-aplicacion"

Configuración de Base de Datos​

PostgreSQL (Recomendado)​

# Instalar PostgreSQL
# Windows: Descargar desde https://www.postgresql.org/download/windows/
# Linux: sudo apt-get install postgresql postgresql-contrib
# macOS: brew install postgresql

# Crear base de datos
createdb sistema_gestion

# Configurar usuario
psql -c "CREATE USER sistema_user WITH PASSWORD 'tu_password';"
psql -c "GRANT ALL PRIVILEGES ON DATABASE sistema_gestion TO sistema_user;"

MySQL​

# Instalar MySQL
# Windows: Descargar desde https://dev.mysql.com/downloads/mysql/
# Linux: sudo apt-get install mysql-server
# macOS: brew install mysql

# Crear base de datos
mysql -u root -p
CREATE DATABASE sistema_gestion;
CREATE USER 'sistema_user'@'localhost' IDENTIFIED BY 'tu_password';
GRANT ALL PRIVILEGES ON sistema_gestion.* TO 'sistema_user'@'localhost';
FLUSH PRIVILEGES;

Configurar Restauración​

# Script de restauración
#!/bin/bash
# restore.sh

BACKUP_FILE=$1
DATABASE_NAME="sistema_gestion"

if [ -z "$BACKUP_FILE" ]; then
    echo "Uso: ./restore.sh <archivo_backup>"
    exit 1
fi

# Restaurar base de datos
pg_restore -d $DATABASE_NAME $BACKUP_FILE

# Reiniciar servicios
systemctl restart sistema-gestion

Configuración de Despliegue​

Docker Configuration​

# Dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

EXPOSE 3000

CMD ["npm", "start"]

Docker Compose​

# docker-compose.yml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/sistema_gestion
    depends_on:
      - db
      - redis

  db:
    image: postgres:15
    environment:
      - POSTGRES_DB=sistema_gestion
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

volumes:
  postgres_data:

Consejos de Configuración​

Mejores Prácticas​

  1. Seguridad:

    • Usa contraseñas seguras
    • Habilita HTTPS en producción
    • Configura firewall apropiadamente

  2. Rendimiento:

    • Configura caché Redis
    • Optimiza consultas de base de datos
    • Usa CDN para archivos estáticos

  3. Monitoreo:

    • Configura alertas apropiadas
    • Monitorea recursos del sistema
    • Revisa logs regularmente

  4. Backup:

    • Configura respaldos automáticos
    • Prueba restauraciones regularmente
    • Almacena respaldos en múltiples ubicaciones
